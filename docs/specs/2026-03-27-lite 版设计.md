## 一、系统定位

**个人盯盘工具** - 极简、快速部署、专注于 Pinbar 信号推送

| 维度 | 规格 |
|------|------|
| **目标用户** | 个人交易者 |
| **核心场景** | 实时监控股币合约，Pinbar 形态出现时推送飞书通知 |
| **部署方式** | Docker Compose 一键启动 |
| **运维成本** | 零配置，启动后无需干预 |

## 四、核心逻辑

### 数据流

```
┌─────────────────────────────────────────────────────┐
│                  Lite 信号管道                        │
├─────────────────────────────────────────────────────┤
│                                                      │
│  Binance WebSocket                                   │
│       ↓                                              │
│  K 线流 (15m + 1h 双周期)                            │
│       ↓                                              │
│  ┌──────────────────────────────────────┐           │
│  │  顺大逆小判断                         │           │
│  │  ├── 1h EMA 趋势判定 (多头/空头)       │           │
│  │  │    └── EMA(period=60)              │           │
│  │  └── 15m Pinbar 检测 (顺大方向)        │           │
│  │       ├── 看涨 Pinbar: 长下影线        │           │
│  │       └── 看跌 Pinbar: 长上影线        │           │
│  └──────────────────────────────────────┘           │
│       ↓                                              │
│  信号生成 (方向 + 入场价 + 止损价)                    │
│       ↓                                              │
│  飞书 Webhook 推送                                    │
│       ↓                                              │
│  日志记录 (lite.log)                                 │
│                                                      │
└─────────────────────────────────────────────────────┘
```

### 顺大逆小逻辑

```python
# 1h 周期 - EMA 趋势判定
if close_1h > ema_1h:
    big_trend = "Bullish"   # 多头趋势
elif close_1h < ema_1h:
    big_trend = "Bearish"   # 空头趋势

# 15m 周期 - Pinbar 检测（仅顺大方向）
if big_trend == "Bullish":
    if is_bullish_pinbar(kline_15m):  # 看涨 Pinbar
        fire_signal("LONG", kline_15m)
elif big_trend == "Bearish":
    if is_bearish_pinbar(kline_15m):  # 看跌 Pinbar
        fire_signal("SHORT", kline_15m)
```

### Pinbar 判定规则

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `min_wick_ratio` | 0.6 | 影线占比下限 |
| `max_body_ratio` | 0.3 | 实体占比上限 |
| `body_position_tolerance` | 0.1 | 实体位置容差 |

**看涨 Pinbar**:
- 下影线长度 / K 线总长度 >= 0.6
- 实体长度 / K 线总长度 <= 0.3
- 实体位于 K 线顶部 10% 区间

**看跌 Pinbar**:
- 上影线长度 / K 线总长度 >= 0.6
- 实体长度 / K 线总长度 <= 0.3
- 实体位于 K 线底部 10% 区间

## 五、配置设计

### 配置文件 (config.yaml)

```yaml
# 交易所配置
exchange:
  name: binance
  api_key: "YOUR_API_KEY"
  api_secret: "YOUR_API_SECRET"
  testnet: false  # true = 测试网，false = 实盘

# 监控币种
symbols:
  - BTC/USDT:USDT
  - ETH/USDT:USDT
  - SOL/USDT:USDT

# 监控周期
timeframes:
  - 15m
  - 1h

# 策略参数
strategy:
  ema_period: 60
  pinbar:
    min_wick_ratio: 0.6
    max_body_ratio: 0.3

# 通知配置
notification:
  feishu_webhook: "https://open.feishu.cn/open-apis/bot/v2/hook/..."
```

### 环境变量 (可选)

```bash
# .env
EXCHANGE_API_KEY=your_api_key
EXCHANGE_API_SECRET=your_api_secret
FEISHU_WEBHOOK=your_webhook_url
```

## 七、部署设计

### Dockerfile

```dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["python", "lite.py"]
```

### docker-compose.yml

```yaml
version: '3.8'

services:
  lite:
    build: .
    container_name: dingpang-lite
    volumes:
      - ./config.yaml:/app/config.yaml
      - ./logs:/app/logs
    restart: unless-stopped
    networks:
      - lite-network

networks:
  lite-network:
    driver: bridge
```

### 启动命令

```bash
# 方式 1: Docker Compose (推荐)
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止
docker-compose down

# 方式 2: 本地运行
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python lite.py
```

## 九、错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| WebSocket 断线 | 指数退避重连 (1s → 60s) |
| API 密钥无效 | 启动时校验，失败退出 |
| 飞书推送失败 | 记录日志，继续运行 |
| K 线数据异常 | 跳过该 K 线，记录警告 |
| 配置缺失 | 启动时校验，失败退出 |
